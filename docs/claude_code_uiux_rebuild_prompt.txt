================================================================================
0. KONTEKS BESAR
================================================================================

Kamu sedang mengerjakan project web VANGUARD di folder:
/Users/vdr/Documents/GPS

Project ini adalah aplikasi fleet monitoring / GPS tracking / task monitoring yang terinspirasi dari TRAMOS Systems.

TRAMOS adalah web tracking monitoring safety untuk armada kendaraan. Secara garis besar, TRAMOS dipakai oleh dispatcher, supervisor, admin transport, dan tim safety untuk:
- Memantau kendaraan realtime di peta.
- Melihat status kendaraan: driving, idle, parking, offline, delayed update.
- Mengelola dan memantau task pengiriman.
- Melihat route yang sudah dilalui driver dan route yang sudah ditentukan.
- Mengecek notification/alarm seperti speeding, GPS delayed, geofence event, camera event.
- Membuka history perjalanan.
- Membuat report trip, idle, stop, speeding, working time, fuel, geofence.
- Melihat camera snapshot dan dashcam monitor.
- Mengelola unit, driver, user, alert, webhook, dan database melalui control panel.

Dokumen knowledge base TRAMOS sudah ada di:
docs/tramos_knowledge_base.txt

Baca dokumen itu sebagai referensi domain, istilah, modul, dan cara kerja TRAMOS. Namun jangan copy desain TRAMOS secara mentah karena desain asli TRAMOS terasa tua. Ambil struktur fitur dan pola operasionalnya, lalu buat VANGUARD lebih modern, jelas, rapi, dan demo-ready.

================================================================================
1. MASALAH PROJECT SAAT INI
================================================================================

Project sudah punya banyak halaman, tapi UI/UX masih perlu disempurnakan besar-besaran.

Masalah utama:
1. Banyak komponen terasa berantakan dan tidak konsisten.
2. Banyak button terlihat bisa diklik tetapi belum punya action yang jelas.
3. Beberapa fitur overlap / double fungsi.
4. Struktur menu belum sepenuhnya menjelaskan perbedaan antar halaman.
5. Beberapa halaman masih terasa seperti placeholder, bukan aplikasi operasional beneran.
6. Ada page yang kontennya kurang lengkap dibanding kebutuhan sistem tracking seperti TRAMOS.
7. Layout table/map/detail panel harus dibuat lebih ergonomis untuk dispatcher.
8. Design harus putih, jelas, maskulin, fleet/logistics, modern, tidak ramai, tidak terlalu “AI-generated”.
9. Dark mode tetap harus ada dan rapi.
10. Semua page di sidebar harus bisa dibuka, punya konten, dan semua action utama harus merespon.
11. Jangan ada menu mati seperti href "#", kecuali memang jelas disabled dengan label "Soon"; lebih baik implement minimal page/settings agar tidak ada dead menu.

- Fleet control tower.
- Logistics / truck / GPS / monitoring.
- Maskulin, tegas, bersih.

- Topbar ringkas.

- Dashboard harus ringkas, bukan landing page.
- Jangan pakai hero besar.
- Jangan buat card bertumpuk di dalam card.
- Jangan terlalu banyak grid kecil yang membuat mata capek.
- Table harus readable, dense, dan bisa discan.

Typography:

Komponen:
- Button harus jelas primary/secondary/danger.
- Icon gunakan lucide-react yang sudah ada.
- Segmented control untuk Table/Map.
- Toggle untuk layer map.
- Search/filter jelas.
- Status badge konsisten.
- Empty state harus ada.
- Loading state harus ada.
- Error/degraded state harus ada.

TRAMOS bukan sekadar dashboard biasa. TRAMOS adalah operational monitoring system.

   - Driving.
   - Idle.
   - Parking/Stop.

   - Delayed update.
   - No GPS data.
   - No RFID Driver.

2. Realtime Monitor

   Fitur:

   - Search unit/driver.
   - Filter status.
   - Vehicle list.
   - Marker kendaraan.
   - Detail unit.
   - Status GPS/websocket.
   - Layer control.
   - Event/notification.

3. Task Monitor / TMS

   Table view:
   - No.
   - Vehicle.
   - Driver.
   - Group.
   - Time.
   - Trip Type.
   - Status.
   - Task.
   - Task Ref.
   - Trip.
   - Origin.
   - Destination.
   Map view:
   - Peta.
   - Route yang sudah dilalui driver.
   - Route yang sudah ditentukan/planned.
   - Marker origin, current vehicle, destination.
   - Panel kanan berisi state, speed, task, schedule, trip, type, origin, destination, distance, traveled, estimated.
   - Notification speeding di bawah sebagai alert, bukan bagian route.
   Toolbar:
   - Table/Map.
   - Order By.
   - Refresh.
   - History.
   - Manage.
   - Customize Column.
   - Zoom to All.
   - Ungroup Vehicles.

4. History
   Untuk melihat perjalanan masa lalu:
   - Unit/driver selector.
   - Date range.
   - Map replay.
   - Timeline.
   - Detail.
   - Engine.
   - Driving.
   - Idle.
   - Stop.
   - Speeding.
   - Event.
   - Reverse.
   - Geofence.
   - Export CSV.

5. Report
   Untuk laporan operasional:
   - Trip Summary.
   - Idle Report.
   - Stop Report.
   - Working Time.
   - Reverse Summary.
   - DIMS / Driver Improvement.
   - Monthly Summary.
   - Fatigue Summary.
   - Mileage Day.
   - Alarm Video.
   - Speeding.
   - Fuel Usage.
   - Geofence Events.

6. Camera Snapshot / PhotoCam
   Bukti visual event:

   - Fuel stop.

   - Geofence entry/exit.

7. Dashcam Monitor
   Video monitor:
   - Channel front/cabin/rear.

   - Event list.

8. Geofence
